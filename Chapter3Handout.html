៣.២ & ៣.៣ ថ្នាក់គ្រឹះ Controller/ControllerBase, IActionResult, and Data Passing
Action Method នៅក្នុង Controller ដំណើរការ Logic និងសម្រេចចិត្តបញ្ជូនទិន្នន័យត្រឡប់ទៅវិញជាប្រភេទ IActionResult។ សម្រាប់ MVC Controller យើងប្រើ helper method View() ដើម្បីហៅទៅ Razor View មកបង្ហាញ។ សម្រាប់ API Controller យើងប្រើ helper method Ok(data) ដើម្បីបញ្ជូនទិន្នន័យចេញជាទម្រង់ JSON។
ប្រភេទ IActionResult និង HTTP Status Codes សំខាន់ៗ
ប្រភេទលទ្ធផល HTTP Status អត្ថន័យបច្ចេកទេស និងការប្រើប្រាស់ជាក់ស្តែង
View(model) 200 OK (MVC) បញ្ជូនទិន្នន័យទៅឱ្យឯកសារ Razor View (.cshtml) ដើម្បីបង្ហាញជា HTML ស្អាត។
Ok(data) 200 OK (API) សំណើជោគជ័យ និងបញ្ជូនទិន្នន័យត្រឡប់ជា JSON format ទៅ Client។
BadRequest() 400 Bad Request ទម្រង់សំណើផ្ញើមកខុស ឬមិនពេញលេញ។ ប្រើប្រាស់នៅពេល Client បញ្ជូនទិន្នន័យមកខុសលក្ខខណ្ឌ។
NotFound() 404 Not Found រកមិនឃើញទិន្នន័យដែលស្នើសុំ។ ប្រើប្រាស់ពេលស្វែងរកទិន្នន័យតាម ID រួចរកមិនឃើញក្នុង Database។

ពន្យល់ពាក្យគន្លឹះ (Glossary)
ពាក្យបច្ចេកទេស និងការពន្យល់ន័យជាភាសាខ្មែរសម្រាប់ជំពូកទី ៣ នេះ៖
ពាក្យបច្ចេកទេស ការពន្យល់ជាភាសាខ្មែរ
Razor View (.cshtml) ឯកសារទំព័រវេបសាយ ដែលរួមបញ្ចូលគ្នារវាងកូដ HTML ធម្មតា និងកូដ C# ដើម្បីបង្ហាញទិន្នន័យ។
Strongly-Typed View ការចងភ្ជាប់ទំព័រ View ទៅនឹង Class Model ជាក់លាក់មួយ ដើម្បីធានាសុវត្ថិភាព និងល្បឿនសរសេរកូដ។
ViewBag / ViewData យន្តការស្តុកទុកទិន្នន័យបណ្តោះអាសន្ន ដើម្បីបញ្ជូនពី Controller ទៅកាន់ View (មិនទាមទារ Class Model ឡើយ)។
ControllerBase Class គ្រឹះរបស់ ASP.NET Core សម្រាប់បង្កើត Web API (មិនមានផ្ទុកយន្តការ render HTML View)។
System.Text.Json បណ្ណាល័យបច្ចេកវិទ្យាក្នុង .NET សម្រាប់បំប្លែង C# Objects ទៅជា JSON (Serialization) និងច្រាសមកវិញ។
Conventional Routing ការកំណត់លំនាំ URL Route ជារួមសម្រាប់គម្រោងទាំងមូល (ពេញនិយមប្រើសម្រាប់គម្រោង MVC standard)។
Attribute Routing ការកំណត់ផ្លូវ URL route ផ្ទាល់នៅលើក្បាល Class ឬ Action Methods (ពេញនិយមបំផុតសម្រាប់ Web API)។
IActionResult Interface កំណត់លទ្ធផលត្រឡប់ការងាររបស់ Action ដែលអនុញ្ញាតឱ្យបញ្ជូនទាំងទិន្នន័យ និង HTTP Status Code។

សេចក្តីសង្ខេបជំពូកទី ៣ (Summary) & References
  • Controllers: ជាខួរក្បាល និងជាអ្នកសម្របសម្រួលលំហូរការងារដ៏សំខាន់សម្រាប់ទាំងគម្រោង MVC (Views) និង Web API (JSON)។
  • MVC (Views) VS API (JSON)៖ MVC បញ្ជូនទំព័រ HTML Render ពី Server តាមរយៈ Controller រីឯ API បញ្ជូនទិន្នន័យសុទ្ធជា JSON តាមរយៈ ControllerBase។

  • Serialization ស្វ័យប្រវត្ត៖ ASP.NET Core ប្រើប្រាស់ System.Text.Json ដើម្បីបំប្លែងទិន្នន័យ C# objects ទៅជា JSON Response ដោយស្វ័យប្រវត្ត។

១. សំណួរ ខុស/ត្រូវ (True/False)

សេចក្តីណែនាំ៖ សូមគូសសញ្ញា (ត្រូវ) ប្រសិនបើប្រយោគខាងក្រោមត្រឹមត្រូវ ឬសញ្ញា (ខុស) ប្រសិនបើខុស។

  1. Web API Controllers របស់ ASP.NET Core គួរតែស្នង (Inherit) ពី class ControllerBase ជានិច្ច។
  2. Attribute [ApiController] ជួយបំពេញការងារត្រួតពិនិត្យ validation ទិន្នន័យដោយស្វ័យប្រវត្ត។
  3. យន្តការ ViewBag និង ViewData ផ្តល់នូវសុវត្ថិភាពទិន្នន័យ (Type Safety) ល្អបំផុតសម្រាប់ Razor Views។
  4. យន្តការ Ok(data) បញ្ជូន Status Code 200 OK ត្រឡប់ទៅកាន់ Client។
  5. Attribute Routing គឺជាការសរសេរ Route URLs នៅលើក្បាល class ឬ method របស់ Controller ផ្ទាល់។
  6. បណ្ណាល័យ System.Text.Json បំប្លែង C# Objects ទៅជា JSON Response ដោយស្វ័យប្រវត្ត។
  7. សញ្ញា [] នៅក្នុងទម្រង់ JSON ត្រូវបានប្រើប្រាស់សម្រាប់សម្គាល់ Object នីមួយៗ។
  8. Conventional Routing ត្រូវបានណែនាំឱ្យប្រើប្រាស់ជាចម្បងក្នុងគម្រោង Web API របស់ .NET។
  9. លទ្ធផល Http Status Code 404 សម្គាល់ថា រកមិនឃើញធនធានទិន្នន័យដែលបានស្នើសុំនៅលើ Server។
  10. IActionResult អនុញ្ញាតឱ្យយើងបញ្ជូនត្រឡប់ទាំងទិន្នន័យ និង Http Status Code ក្នុងពេលតែមួយ។
