១. ការពិពណ៌នាមុខវិជ្ជា (Course Description)
មុខវិជ្ជានេះត្រូវបានរចនាឡើងយ៉ាងពិសេសសម្រាប់និស្សិតដែលទើបឈានជើងចូលក្នុងពិភពនៃការអភិវឌ្ឍប្រព័ន្ធ Back-End និង Application Programming Interface (API) កម្រិតដំបូង។ នៅក្នុងយុគសម័យឌីជីថល កម្មវិធីវិប (Web Application) និងទូរស័ព្ទដៃ (Mobile Application) សុទ្ធតែពឹងផ្អែកលើ Application Programming Interface (API) ជាស្ពានចម្លងទិន្នន័យ។ យើងផ្តោតសំខាន់លើស្ថាបត្យកម្មស្តង់ដារ Model-View-Controller (MVC)។ និស្សិតមិនត្រឹមតែចេះកំណត់រចនាសម្ព័ន្ធ Model (មូលដ្ឋានទិន្នន័យ - Database) និង Controller (API Logic) ប៉ុណ្ណោះទេ តែតម្រូវឱ្យអនុវត្តសរសេរកូដ Frontend ពោលគឺ HyperText Markup Language (HTML) និង JavaScript (JS) ដែលដើរតួជា View តាមទម្រង់ Model-View-Controller (MVC) ដើម្បីហៅទិន្នន័យពី API មកបង្ហាញ និងបញ្ជាផ្ទាល់នៅលើផ្ទាំង User Interface (UI)។
សទ្ទានុក្រម និងពាក្យគន្លឹះ (Key Terms)

Model-View-Controller (MVC)

ស្ថាបត្យកម្មសរសេរកូដបែងចែកកម្មវិធីជា ៣ ផ្នែកធំៗ៖ Model (ទិន្នន័យ), View (ផ្ទាំងបង្ហាញ), Controller (អ្នកចាត់ចែង)។

Entity Framework Core

បច្ចេកវិទ្យា Object-Relational Mapping (ORM) ដែលជួយឱ្យ Model អាចតភ្ជាប់ទៅ Database (DB) បានយ៉ាងងាយតាមវិធីសាស្ត្រ Code-First។

CRUD Operations

ប្រតិបត្តិការគោលទាំង៤ នៃប្រព័ន្ធទិន្នន័យ៖ Create (បញ្ជូល), Read (អាន), Update (កែប្រែ), Delete (លុប)។

CORS Policy

Cross-Origin Resource Sharing (CORS): គោលការណ៍សុវត្ថិភាពអនុញ្ញាតឱ្យ Frontend អាចហៅ Application Programming Interface (API) ឆ្លង Domain ផ្សេងគ្នាបាន។

២. លទ្ធផលសិក្សារំពឹងទុក (Course Learning Outcomes - CLO)

ក្រោយបញ្ចប់ការសិក្សាលើមុខវិជ្ជានេះ និស្សិតនឹងទទួលបានសមត្ថភាព និងលទ្ធផលរំពឹងទុកដូចខាងក្រោម៖

លទ្ធផលសិក្សានៃមុខវិជ្ជា (CLO) លទ្ធផលនៃកម្មវិធីអប់រំ​ (PLO)
CLO 1 (ស្ថាបត្យកម្ម API និង C#) រៀបចំ Environment ដំឡើងកម្មវិធី Visual Studio Code (VS Code), .NET Software Development Kit (SDK) និងយល់ដឹងពីស្ថាបត្យកម្មគ្រឹះនៃ ASP.NET Core រួមទាំងការសរសេរកូដភាសា C# កម្រិតមូលដ្ឋាន។
CLO 2 (ការគ្រប់គ្រង Controller និង Route) អនុវត្តការបង្កើត Controllers តាមទម្រង់ Model-View-Controller (MVC), ការកំណត់ Routing និងការប្រើប្រាស់ Hypertext Transfer Protocol (HTTP) Methods សម្រាប់ទទួលសំណើពី Client ប្រកបដោយភាពត្រឹមត្រូវ។
CLO 3 (EF Core & SQL Server) រចនា និងគ្រប់គ្រងមូលដ្ឋានទិន្នន័យ (Database) តាមរយៈបច្ចេកវិទ្យា Entity Framework Core (EF Core) ដោយប្រើវិធីសាស្ត្រ Code-First Migrations ភ្ជាប់ជាមួយកម្មវិធី SQL Server Management Studio (SSMS)។
CLO 4 (ប្រតិបត្តិការ CRUD Full-Stack) បង្កើតមុខងារ Create, Read, Update, Delete (CRUD) យ៉ាងស្ទាត់ជំនាញ ទាំងកម្រិត Backend Application Programming Interface (API) និងការបង្កើត User Interface (UI) តភ្ជាប់តាមរយៈ JavaScript (JS) Fetch API។
CLO 5 (ការអនុវត្តគម្រោង និង GitHub) ដោះស្រាយបញ្ហាសុវត្ថិភាព Cross-Origin Resource Sharing (CORS), អភិវឌ្ឍន៍ Representational State Transfer (RESTful) API ពេញលេញ និងប្រើប្រាស់ប្រព័ន្ធ GitHub សម្រាប់គ្រប់គ្រងកូដគម្រោងជាក្រុម។

១ វត្តមាន​ (Attendance) ១០% ការចូលរួម និងសកម្មភាពក្នុងថ្នាក់រៀន។
២ កិច្ចការផ្ទះ និងស្រាវជ្រាវ (Homework and Assignment) ៣០% កិច្ចការសរសេរកូដ C# ការបង្កើត Application Programming Interface (API) & Web Page User Interface (UI) និងការរុញកូដ (Push) ទៅកាន់ GitHub Repository។
៣ ប្រឡងពាក់កណ្តាលឆមាស (Mid-Term Exam) ២០% ប្រឡងពាក់កណ្ដាលឆមាស (ផ្តោតលើរៀបចំ Database និងទាញទិន្នន័យបង្ហាញលើ Web Page)។
៤ ប្រឡងបញ្ចប់ឆមាស (Final Exam) ៤០% ប្រឡងបញ្ចប់ឆមាស (ធ្វើប្រតិបត្តិការ Create, Read, Update, Delete (CRUD) API និង Web Page ពេញលេញ)។

៥.១- រង្វាយតម្លៃលើវត្តមាន និងការចូលរួម

និស្សិតត្រូវបានណែនាំឲ្យចូលរៀនទាន់ពេលវេលា។ និស្សិតចូលរៀនយឺតចន្លោះពី ១០នាទី ទៅ ២០នាទី នៃ session នីមួយៗ ត្រូវចាត់ទុកថាអវត្តមានចូលរៀនយឺតដែលសម្គាល់ដោយ (AL)។ ករណីនិស្សិតវត្តមានចូលរៀនយឺត (AL) ចំនួន៣ដង ស្មើនឹងនិស្សិតអវត្តមាន (A) មួយ session។ និស្សិតរៀនយឺត​លើស​ពី ២០នាទី មិនត្រូវអនុញ្ញាតឲ្យចូលរៀនក្នុងថ្នាក់ឡើយ ប៉ុន្តែអាចសិក្សាស្រាវជ្រាវក្នុងបណ្ណាល័យ​។ និស្សិតត្រូវបានសាស្ត្រាចារ្យស្រង់វត្តមាននៅរៀងរាល់ដើម session នីមួយៗ។

បច្ចេកវិទ្យា ASP.NET និងប្រព័ន្ធគ្រប់គ្រងមូលដ្ឋានទិន្នន័យ SQL Server

.NET Core MVC

៥.២- កិច្ចការស្រាវជ្រាវ (Group Assignment) និងការប្រើប្រាស់ GitHub

  • – កាលបរិច្ឆេទចុងសប្តាហ៍ទី ១១ ដាក់ពិន្ទុគឺ ១៥ % (តម្រូវឱ្យមានការ Push កូដចូល GitHub ជាប្រចាំដើម្បីវាយតម្លៃការចូលរួមរបស់សមាជិកម្នាក់ៗ)
  • – ប្រធានបទដែលដាក់ឲ្យពួកគាត់ស្រាវជ្រាវ (គម្រោង Full-Stack Model-View-Controller (MVC) សាមញ្ញ)៖
    1. School & Student Management: រចនា Database (DB), សរសេរ Application Programming Interface (API) សម្រាប់ចាត់ចែងទិន្នន័យ (Model & Controller) និងបង្កើតផ្ទាំង Web Page (View) ដើម្បីអនុញ្ញាតឱ្យរដ្ឋបាលសាលាបញ្ចូល និងកែប្រែទិន្នន័យបាន។
    2. E-Commerce Product Catalog: រចនា API សម្រាប់គ្រប់គ្រងផលិតផល (Products) ដែលមានទិន្នន័យឈ្មោះ និងតម្លៃ។ បង្កើត Web Page User Interface (UI) សម្រាប់ឱ្យអតិថិជនមើលមុខទំនិញ និងម្ចាស់ហាងអាចកែប្រែទំនិញបានផ្ទាល់។
    3. Task Manager Application: កសាង API ដើម្បីកត់ត្រាកិច្ចការប្រចាំថ្ងៃ (To-Do List)។ បង្កើត Web Page UI ដើម្បីឱ្យអ្នកប្រើប្រាស់អាចបញ្ចូលការងារថ្មី និងចុច (Tick) ប្តូរស្ថានភាពការងារថាធ្វើរួចរាល់។
  • – ភារកិច្ចនិស្សិត៖ ធ្វើការជាក្រុម (២-៣ នាក់), សរសេរកូដ ASP.NET Core API រួមជាមួយការសរសេរ HyperText Markup Language (HTML) និង JavaScript (JS) សម្រាប់ Web Page តាមទម្រង់ Model-View-Controller (MVC) និងឡើងធ្វើបទបង្ហាញគម្រោងទាំងមូលក្នុងថ្នាក់។

៥.២.២- កិច្ចការពិភាក្សាជាក្រុមតូចៗនៅក្នុងថ្នាក់ (Group discussion)

  • · កាលបរិច្ឆេទ ៖ អនុវត្តជាប្រចាំ។ ដាក់ពិន្ទុគឺ ៥ % (យកពីចំណុចចូលរួមក្នុងថ្នាក់)
  • · ប្រធានបទ ៖ ពិភាក្សាពីភាពខុសគ្នារវាងការធ្វើការលើ Backend Application Programming Interface (API) និងការសរសេរកូដ Frontend (Web Page), បញ្ហា Cross-Origin Resource Sharing (CORS) ពេលហៅ API ឆ្លង Port, និងការដោះស្រាយបញ្ហា Conflict កូដពេលតភ្ជាប់ចូលគ្នាលើ GitHub។

៥.២.៣- សំណួរសរសេររយៈពេល១៥នាទី

  • · កាលបរិច្ឆេទ ៖ ធ្វើឡើង ៣ ទៅ ៤ ដង ក្នុងឆមាស (Surprise Quiz) ដាក់ពិន្ទុ ១០ %
  • · ប្រធានបទ ៖ ការប្រកាសអញ្ញាត C# ងាយៗ, ការប្រើប្រាស់ .NET Command Line Interface (CLI) Commands (dotnet run), ការសរសេរ Route លើ Controller, និងកូដ JavaScript (JS) Fetch API មូលដ្ឋាន។

៥.៣- ប្រឡងពាក់កណ្តាលឆមាស

  • · កាលបរិច្ឆេទ ៖ ចុងសប្តាហ៍ទី ៩ ដាក់ពិន្ទុ ២០ %
  • · ប្រធានបទ ៖ គ្របដណ្តប់ជំពូកទី ១ ដល់ទី ៨ (ផ្តោតលើការរៀបចំ Model, ការទាញទិន្នន័យ GET ពី API, និងការសរសេរ Fetch API បង្ហាញលើ Web Page)។ រយៈពេល ៩០នាទី ។

៥.៤- ប្រឡងបញ្ចប់ឆមាស

  • · ផ្តល់ព័ត៌មានដោយការិយាល័យសិក្សា ដាក់ពិន្ទុ ៤០ %
  • · ប្រធានបទ ៖ គ្របដណ្តប់មេរៀនទាំងមូល (ការធ្វើ Create, Read, Update, Delete (CRUD) API និងភ្ជាប់ជាមួយ Web Page ពេញលេញ)។ រយៈពេល ១២០ នាទី។
